Vuex是什么

​ 是一个专为cue.js应用程序开发的状态管理模式,集中式存储管理,解决多组件通信

特点:

  1. vue官方搭配，专属使用 (类似于：vue-router)，有专门的调试工具
  2. 集中式管理数据状态方案 (操作更简洁)data() { return { 数据, 状态 }}
  3. 数据变化是可预测的 (响应式)

​

Vuex的主要内容:

  • state： 统一定义公共数据（类似于data(){return {a:1, b:2，xxxxxx}}）
  • mutations ： 使用它来修改数据(类似于methods)===更改state中的数据必须使用mutations来进行更改
  • getters： 类似于computed(计算属性，对现有的状态进行计算得到新的数据——-派生 )
  • actions： 发起异步请求
  • modules： 模块拆分

组件

什么是组件

​ 组件是一个可复用的vue实例对象,封装了HTML,css,js

为什么要学组件?

​ 方便开发,便于复用,容易维护

​ 什么时候用封装组件:遇到可重复标签.可复用的时候

组件的基础

组件使用步骤:

​ 1.创建组件 -文件名.vue

​ 2,引入组件

​ 3,注册组件(全局) vue.component(‘组件名”,组件对象)或,局部组件

​ 4.使用组件名

scoped原理:

​ 1.当前组件的所有的标签加data-v-hash

​ 2.当前所有的css选择器加 属性选择器[data-v-hash]

组件通信

(父传子):

  1. 子组件中定义 props 来接收父组件传递过来的数据

  2. 父组件中给子组件标签上进行属性绑定传值

    循环组件 v-for:

    可以写在组件标签上,循环创建组件

    v-bind动态绑定属性给子组件传值

    动态属性

    ​ 子组件改符传入的数据不通知父级,数据的不变

    2.props 是只读 不能修改

    组件通信_单向数据流:

    ​ 从父到子的数据流向,叫单向数据流

(子传父):

​ 子组件触发父的自定义事件

​ 子组件内,恰当的实际,触发付给我绑定的自定义事件,大致父级 methods 里事件处理函数执行

​ this.$emit(‘父组件事件名’,索引,)

流程:

​ 1.父组件中给子组件绑定自定义事件.定义时间处理函数

​ 2.子组件在恰当时机触发事件

​ 3.恰当时机是这用户点击按钮的时候

​ 4使用 this.$emit()触发自定义事件

​ 5传递数据给父组件

​ 6父组件事件处理函数触发.接收数据并处理

Vue基础API_计算属性_过滤器_侦听器:

能改原始的数组或将数组翻转可以让v-for更新

vue无法检测到直接通过索引更改元素的情况,页面不会刷新

push(),pop(),shift(),unshift(),splice(),sort(),reverse()会导致更新

filter().concat().slice() 返回新数组.不会导致v-for的更新 可以采取覆盖掉

this.$set(目标结构,更新位置,更新的值)可以更新

虚拟dom 一个js对象 保存DOM关键信息

v-for更新的时候,如何操作DOM?

循环新的DOM结构.和旧的DOM结构对比.尝试就地复用原有的标签,更新内容

新旧DOM的对比,提高性能,不用频繁操作DOM

template 转成虚拟dom再转为dom

根元素变了 删除并更新

属性变了 只更新属性

无key 就地更新内容

有key 按照key比较

key只能是惟一的的字符串或者数值

有id用id,无id用索引

配合虚拟DOM+diff算法提高更新的性能

过滤器:

只能用在插值表达式和v-bind动态属性里

语法:vue.filter(过滤器名(值)=>{返回处理后的函数}(全局)

filters:{过滤器名(值)=>{返回处理后的函数}(局部)

本质就是一个函数

watch 监听器:

当数据变化时触发函数调用,将变化后的数据和变化前的数据传递过来

监听的名字必须和监听数据名字一样

语法 watch {

}

}

深度监听:

‘属性名’:{

重绘:当页面的颜色发生改变的时候会大致重绘

性能的影响:回流比重绘更消耗性能
